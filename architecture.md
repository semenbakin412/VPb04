Классы системы
User (Пользователь)
Атрибуты:
id — уникальный идентификатор пользователя (генерируется автоматически);
username — имя пользователя (логин);
full_name — полное имя пользователя;
email — электронная почта пользователя;
created_at — дата и время регистрации пользователя (datetime, устанавливается автоматически);
projects — список проектов пользователя (list объектов класса Project).
Методы:
create_project(title, description="") — создаёт новый проект и добавляет его в список projects, возвращает созданный объект Project;
get_project_by_id(project_id) — ищет проект по ID и возвращает его (или None, если не найден);
get_all_tasks() — возвращает список всех задач пользователя из всех проектов;
get_overdue_tasks() — возвращает список просроченных задач (у которых due_date меньше текущей даты и is_done = False);
get_tasks_by_priority(priority) — возвращает задачи с указанным приоритетом.
Project (Проект)
Атрибуты:
id — уникальный идентификатор проекта (генерируется автоматически);
title — название проекта;
description — описание проекта;
created_at — дата и время создания проекта (datetime);
tasks — список задач проекта (list объектов класса Task).
Методы:
add_task(task) — добавляет задачу в проект;
remove_task(task_id) — удаляет задачу из проекта по её ID (если задача не найдена, выводит сообщение «Задача не найдена»);
get_tasks_by_status(is_done) — возвращает список задач с указанным статусом выполнения (True/False);
get_task_by_id(task_id) — ищет задачу по ID внутри проекта и возвращает её (или None, если не найдена).
Task (Задача)
Атрибуты:
id — уникальный идентификатор задачи (генерируется автоматически);
title — название задачи (краткое описание);
description — подробное описание задачи;
is_done — флаг выполнения задачи (False по умолчанию);
priority — приоритет задачи ('Low', 'Medium', 'High', по умолчанию 'Medium');
due_date — дата и время выполнения задачи (datetime, может быть None);
created_at — дата и время создания задачи (datetime, устанавливается автоматически);
completed_at — дата и время завершения задачи (datetime, None, пока is_done = False).

Методы:
mark_as_done() — отмечает задачу как выполненную (is_done = True), устанавливает completed_at = datetime.now();
change_description(new_description) — изменяет описание задачи;
set_priority(new_priority) — устанавливает новый приоритет (проверяет, что приоритет из допустимого списка);
update_due_date(new_due_date) — обновляет дату выполнения;
reopen() — снимает отметку о выполнении (is_done = False, completed_at = None).

Взаимодействие объектов
Один User может иметь несколько Project.
Один Project содержит несколько Task.
Каждая Task принадлежит ровно одному Project.
Связь между объектами:
Объект User хранит список своих проектов (projects).
Объект Project хранит список своих задач (tasks).
Объект Task не хранит ссылку на родительский проект — связь односторонняя (от проекта к задачам).
